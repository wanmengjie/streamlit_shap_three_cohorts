# 主流程变量类型与预处理说明（预测 CPM · 因果 · 与代码对齐）

本文档与 `utils/charls_feature_lists.py`、`utils/charls_sklearn_preprocess_pipelines.py`、`utils/charls_train_only_preprocessing.py`、`modeling/charls_model_comparison.py`、`causal/charls_recalculate_causal_impact.py` 当前实现一致。若修改 `CONTINUOUS_FOR_SCALING` / `ORDINAL_COUNT_IMPUTE_ONLY` / `CATEGORICAL_NO_SCALE`，请同步更新本文。

---

## 一、特征矩阵 X 如何构造

### 1.1 预测打擂（`compare_models`）

1. `exclude = get_exclude_cols(df, target_col=TARGET_COL)`（**不传** `treatment_col`）。
2. `W_cols = [c for c in df.columns if c not in exclude]`。
3. `X = df[W_cols].select_dtypes(include=[np.number])` — **仅数值列**进入模型。

### 1.2 因果主分析（`get_estimate_causal_impact` / XLearner 等）

1. `exclude = get_exclude_cols(df, target_col=Y, treatment_col=T)` — **处理变量 T 被排除**。
2. 同样取**数值列**作为协变量矩阵，再经 `fit_transform_numeric_train_only`（见第三节）。

### 1.3 一律不进 X 的列（基准）

见 `EXCLUDE_COLS_BASE`：`ID`、`wave`、`province`、`communityID`、结局与队列定义相关列、`had_comorbidity_before`、`sleep_adequate`、已废弃/共线列等。

### 1.4 按列名关键字排除（泄露）

`LEAKAGE_KEYWORDS`：列名（小写）含 `cesd`、`total_cog`、`cognition`、`memory`、`executive`、`score`、`test` 的列**全部排除**。  
因此 **CES-D、认知总分等「定义变量」不进预测 X**（避免标签泄漏）。

---

## 二、数值进入 Pipeline 后的两条技术分支

对 **X 中每个数值列**，`build_numeric_column_transformer`（CPM）按名单分为：

| 分支 | 条件 | 预处理（均在 CV 训练折上 fit） |
|------|------|--------------------------------|
| **num** | 列名 ∈ `CONTINUOUS_FOR_SCALING` | **IterativeImputer** → **StandardScaler** |
| **pass** | 其余数值列 | **仅 IterativeImputer** |

**有序计数 / 家庭规模 / ADL·IADL 计分**（`ORDINAL_COUNT_IMPUTE_ONLY`：`adlab_c`、`iadl`、`family_size`）**不在** `CONTINUOUS_FOR_SCALING` 内 → 走 **pass**，**不**做 StandardScaler。

---

## 三、因果协变量矩阵的预处理（与 CPM 名单一致、实现不同）

函数：`utils/charls_train_only_preprocessing.fit_transform_numeric_train_only`。

- **与 CPM 相同**：仍用 `CONTINUOUS_FOR_SCALING` 决定谁做 **StandardScaler**；**`ORDINAL_COUNT_IMPUTE_ONLY`** 中列 **只插补、不缩放**。
- **与 CPM 不同**：
  - 插补为 **`SimpleImputer(strategy='median')`**（**非** `IterativeImputer`）。
  - 在**与 CPM 一致的划分策略**下，仅在**训练子集**上 fit，再 transform 全表。
- **数据层**：因果常用 **`step1_imputed_full` 等完成表**；CPM 常用**保留缺失 + Pipeline 内迭代插补**。见 `docs/MEDICAL_CODE_CAUTIONS.md`。

## 四、变量清单：实质类型 vs 管道处理方式

下列「实质类型」依据 CHARLS/BPS 问卷含义；「管道」指第二节表格。

### 4.1 实质为**近似连续测量** — 管道：**Imputer + StandardScaler**

| 变量 | 说明（摘要） |
|------|----------------|
| `age` | 年龄（岁） |
| `bmi` | 体重指数 |
| `pulse` | 脉搏 |
| `mwaist` | 腰围 |
| `systo` / `diasto` | 收缩压 / 舒张压 |
| `lgrip` | 握力 |
| `wspeed` | 步行速度 |
| `sleep` | 睡眠时长（小时） |
| `income_total` | 预处理中已对非负部分 `log1p`，仍作连续协变量 |

**代码位置**：`CONTINUOUS_FOR_SCALING`。

---

### 4.2 实质为**有序计数 / 家庭规模 / 困难项计分** — 管道：**仅 Imputer，不 StandardScaler**

| 变量 | 说明（摘要） |
|------|----------------|
| `family_size` | 同住家庭人数等，有序整数，**不做** StandardScaler |
| `adlab_c` | 基本 ADL 困难项计数/计分，有限整数档 |
| `iadl` | 工具性 ADL 困难项计数/计分，有限整数档 |

**代码位置**：`ORDINAL_COUNT_IMPUTE_ONLY`（**不在** `CONTINUOUS_FOR_SCALING`）。

**注意**：Table 1（`data/charls_table1_stats.py`）仍将 `family_size` 与 `adlab_c`/`iadl` 等放在 `optional_continuous` 块，表示**描述统计展示分组**，**不等于**建模时做标准化。

---

### 4.3 实质为**二分类（0/1 指示）** — 管道：**仅 Imputer，不 StandardScaler**

问卷或定义上为是否/有无；以数值 0/1 存储。**不得**放入 `CONTINUOUS_FOR_SCALING`（已在 `CATEGORICAL_NO_SCALE` 中声明）。

| 变量 | 说明（摘要） |
|------|----------------|
| `gender` | 性别编码（如 0/1） |
| `rural` | 农村居住 |
| `retire` / `ins` / `pension` | 退休、医保、养老金等 |
| `hibpe` `diabe` `cancre` `lunge` `hearte` `stroke` `arthre` | 各慢病诊断指示 |
| `exercise` `drinkev` | 运动、饮酒（预处理中 exercise 缺失可填 0，与因果 T 一致） |
| `fall_down` `disability` | 跌倒、残疾 |
| `is_socially_isolated` | 社会隔离（衍生规则见预处理） |

**代码位置**：`CATEGORICAL_NO_SCALE`（且 ∉ `CONTINUOUS_FOR_SCALING`）→ **pass 分支**。

---

### 4.4 实质为**多分类 / 有序 Likert**（以整数编码）— 管道：**仅 Imputer，不 StandardScaler**

名义或有序多档，以 1–5 等整数表示。**未**对其实施 One-Hot；树模型直接分裂；线性类模型隐含「等距」近似。

| 变量 | 水平含义（摘要） | 标签见 |
|------|------------------|--------|
| `edu` | 教育程度 1–4 | `CATEGORICAL_LEVEL_LABELS['edu']` |
| `marry` | 婚姻 0/1 等 | `CATEGORICAL_LEVEL_LABELS['marry']` |
| `srh` | 自评健康 1–5 | `CATEGORICAL_LEVEL_LABELS['srh']` |
| `satlife` | 生活满意度 1–5 | `CATEGORICAL_LEVEL_LABELS['satlife']` |

**代码位置**：`CATEGORICAL_NO_SCALE` → **pass 分支**。

---

### 4.5 其他常见数值列（未列入上表时）

- **`smokev`**：当前**不在** `CATEGORICAL_NO_SCALE` 中；若进 X 且不在 `CONTINUOUS_FOR_SCALING`，则走 **pass**（仅 Imputer）。实质多为吸烟指示，与二分类类似。
- **`province`**：在 `EXCLUDE_COLS_BASE` 中 → **不进 CPM/因果主流程 X**（避免高基数地域混杂与划分问题；若子研究需要需单独设计）。

## 五、因果分析特有逻辑（协变量以外）

- **`T.fillna(0)`**：主处理变量（如 `exercise`）缺失按 0。
- **`exercise × adlab_c` 交互项**：当 `T == 'exercise'` 且存在 `adlab_c` 时构造，用于缓解选择偏倚（见 `charls_recalculate_causal_impact.py`）。
