本文档与当前代码（`run_all_charls_analyses.py`、`config.py`、`run_cohort_protocol`、`compare_models` 等）对齐，粒度与「插补用何种方法」同级：**每一步做什么、用什么数据、核心参数在哪、结果写到哪里**。

## 二、原始数据与「两张分析脸」：必须区分

### 2.1 原始数据

- **文件**：根目录 `CHARLS.csv`（`config.RAW_DATA_PATH`）。
- **预处理函数**：`data/charls_complete_preprocessing.preprocess_charls_data`  
  - 年龄下限：`AGE_MIN = 60`  
  - 抑郁截断：`CESD_CUTOFF = 10`（CES-D-10 ≥10 为抑郁）  
  - 认知截断：`COGNITION_CUTOFF = 10`（总分 ≤10 为认知受损）  
  - 写出：`preprocessed_data/CHARLS_final_preprocessed.csv`、`attrition_flow.csv` 等（主流程在需要时会 `write_output=True` 再跑一遍预处理以更新流失图数据）。

### 2.2 表 A：`df_clean`（主分析宽表，因果/多暴露等）

- **当 `USE_IMPUTED_DATA=True` 且 `IMPUTED_DATA_PATH` 存在**：  
  - 读 `imputation_npj_results/pipeline_trace/step1_imputed_full.csv`  
  - `prepare_exposures` → 删 `COLS_TO_DROP` → `reapply_cohort_definition` → 得 **`df_clean`**（**行级一般为完成值**，供因果、扩展分析）。  
- **否则**：`df_clean` 直接来自 `preprocess_charls_data`（**可含缺失**）。

### 2.3 表 B：`df_pre`（预处理、**保留缺失**）

- **仅当**成功走「插补分支」时（`df_base_for_sensitivity` 非空）：主流程会**再跑**一遍 `preprocess_charls_data(..., write_output=True)` 得到 **`df_pre`**，与 `CHARLS.csv` 链一致、**不经过** `step1_imputed_full`。
- **用途**：Table 1、插补敏感性、**以及 CPM/SHAP/临床/外部验证的输入**（见下）。

### 2.4 BMI 等异常时：插补表与预处理表「覆盖合并」

- 若 `df_clean`（插补）中某些连续变量均值落在荒谬区间（如 bmi），而 `df_pre` 合理，则按 `ID`+`wave` 用 **`df_pre` 列覆盖** `df_clean` 对应列（`OVERLAY_RANGES` 逻辑）。

### 2.5 队列切片

- **`df_a / df_b / df_c`**：`df_clean` 按 `baseline_group == 0/1/2`（**因果用**）。  
- **`df_pa / df_pb / df_pc`**：若 `df_pre` 存在，则对 `df_pre` 删 `COLS_TO_DROP` 后 `reapply_cohort_definition` 再分队列（**预测用，保留缺失**）；若 `df_pre` 不存在，则与 `df_a/b/c` 相同（均为 `df_clean` 切片）。

## 三、前置：可选「全样本 bulk 插补」（npj 脚本）

| 配置 | 含义 |
|------|------|
| `RUN_IMPUTATION_BEFORE_MAIN` | `True` 时主流程**先**跑 `archive/charls_imputation_npj_style` 再读 `step1_imputed_full` |
| `USE_IMPUTED_DATA` | `False` 则跳过读插补 CSV，全流程以预处理为准 |

**bulk 层方法（脚本内）**：在完整病例子集上对连续变量人为造 8% MCAR 缺失，用 **NRMSE** 在 6 种方法中选优：**均值、中位数、线性回归、BayesianRidge 型 IterativeImputer、KNN、MissForest**；分类/有序另有规则。产出 **`step1_imputed_full.csv`**，可选 **`step1_imputed_m1..mN`**（`N_MULTIPLE_IMPUTATIONS`）。

**与 CPM 的关系**：CPM **不再**以「仅 bulk 完成表」为唯一输入；预测用 **`df_pre` 队列 + Pipeline 内 `IterativeImputer`**（见 §五）。

## 四、主流程 `main()` 顺序（`MAIN_SKIP_STEPS_BEFORE_COHORTS=False` 时）

下列按**实际执行顺序**排列。

### 4.1 前缀（汇总目录 `OUTPUT_ROOT`，默认 `LIU_JUE_STRATEGIC_SUMMARY`）

1. **概念框架图**  
   - `viz/draw_conceptual_framework.draw_conceptual_framework`  
   - 输出：`fig2_conceptual_framework.png`

2. **Table 1**  
   - `data/charls_table1_stats.generate_baseline_table`  
   - 数据：**`df_for_table1`** = 有 `df_pre` 时用 **`df_pre`**（插补前，STROBE）；否则 `df_clean`  
   - 输出：`table1_baseline_characteristics.csv` 等

3. **纳入排除流程**  
   - 复制 `attrition_flow.csv`，`draw_attrition_flowchart.draw_flowchart`  
   - 写 `attrition_flow_readme.txt`（说明 Table1/流失基于预处理；预测用 Pipeline 插补；因果用 bulk 等）

4. **插补方法敏感性（五种方法与 AUC）**  
   - `evaluation/charls_imputation_audit.run_imputation_sensitivity_preprocessed`  
   - 数据：**`df_for_imp_sens`**（通常与 Table1 同源，为**带缺失**的预处理表）  
   - 输出：`07_sensitivity_imputation/` 下 CSV、图等

### 4.2 三队列核心：`run_cohort_protocol` × 3

对每个 Cohort 传入 **5 元组**：`(df_pred, Cohort_dir, 'A'|'B'|'C', 描述, df_causal)`。

- **`df_pred`**：`df_pa` / `df_pb` / `df_pc`  
- **`df_causal`**：`df_a` / `df_b` / `df_c`（插补后同队列，与因果一致）

**并行**：`config.PARALLEL_COHORTS=True` 时用 joblib 并行跑多个队列（Windows 上默认常关）。

**续跑**：`RUN_COHORTS_ONLY=['B','C']` 等只跑部分队列；未跑队列从磁盘读 Table2/ATE 恢复汇总。

## 五、单队列 `run_cohort_protocol`（逐步）

目录结构由 `config.COHORT_STEP_DIRS` 定义，例如 `01_prediction`、`02_shap`、`03_causal`…

### 5.1 预测打擂 + CPM（`01_prediction`）

1. **`compare_models(df_sub=df_pred, target_col=TARGET_COL)`**  
   - **目标**：`TARGET_COL`（默认 `is_comorbidity_next`），下一波是否发生「抑郁–认知共病」。  
   - **特征**：`utils/charls_feature_lists.get_exclude_cols` 排除泄漏列、干预专用列等后，**仅数值列**进模。  
   - **划分**：  
     - `USE_TEMPORAL_SPLIT=True` 且含 `wave`：**训练 = 所有 wave < max_wave，测试 = max_wave**（样本量不足则退回）  
     - 否则：**`GroupShuffleSplit(test_size=0.2)`，`groups=ID`**（同人不同波不进训练又进测试）。  
   - **预处理 Pipeline**：`utils/charls_sklearn_preprocess_pipelines.build_numeric_column_transformer`  
     - 连续子集：`CONTINUOUS_FOR_SCALING` → **IterativeImputer**（`initial_strategy='median'`，`max_iter=config.ITERATIVE_IMPUTER_MAX_ITER`，`sample_posterior=False`，默认基学习器为 sklearn 的 **BayesianRidge**）→ **StandardScaler**  
     - **有序计数 / 家庭人数 / ADL·IADL 计分**（`adlab_c`、`iadl`、`family_size`，见 `ORDINAL_COUNT_IMPUTE_ONLY`）：**仅 IterativeImputer**，**不** StandardScaler，以保持整数档位语义。  
     - 其余数值列：仅 **IterativeImputer**  
     - **仅在 RandomizedSearchCV 的 GroupKFold 训练折上 fit**  
   - **调参**：每模型 **RandomizedSearchCV**，`cv=GroupKFold(5)`，`scoring='roc_auc'`，`n_iter` 大模型 50、其余 80（见代码）。  
   - **模型池**（最多 15 个）：LR, RF, XGB, GBDT, ExtraTrees, AdaBoost, DT, Bagging, KNN, MLP, NB, SVM, HistGBM, （可选）LightGBM、CatBoost。  
   - **不平衡**：如 `class_weight`、`scale_pos_weight` 等在各自参数网格中搜索。  
   - **阈值（TRIPOD）**：在**训练集**上做 `cross_val_predict` + **Youden 指数**得每模型 `_opt_threshold`，**不在测试集上搜阈**。  
   - **测试集指标 CI**：`get_metrics_with_ci(y_test, y_prob, groups=grp_test)` → **按参与者 ID 的 cluster bootstrap**（1000 次）。  
   - **冠军**：`Recall_raw ≥ 0.05` 子集中 **AUC 最高**；否则退回全模型 AUC 最高。  
   - 输出：`model_performance_full_*.csv`、`multiple_testing_note.txt`、可选 `roc_data.json` 等。

2. **`evaluate_and_report`**（CPM 表）  
   - 用上面各模型的 **测试集** `X_test,y_test` 与 **验证集确定的阈值**，写 **`table2_{cohort}_main_performance.csv`**（含 AUC/Brier/Recall 等及 bootstrap CI）。  
   - Bootstrap 同样支持 **`groups_test`**（cluster bootstrap）。

3. **`_select_champion`**  
   - 从 CPM 表与 `perf_df` 对齐选出冠军模型。

4. **概率校准（可选）**  
   - `CALIBRATE_CHAMPION_PROBA=True`：对冠军 **`CalibratedClassifierCV(method='sigmoid', cv=GroupKFold)`**，仅在 **训练子集** `X_train,y_train,grp_train` 上 fit。

5. **保存冠军**  
   - `champion_model.joblib`、`model_complexity_efficiency.txt`、雷达图等。

### 5.2 SHAP（`02_shap` 等，`pool_only` 时跳过）

- `run_shap_analysis_v2`、`run_stratified_shap`、`run_shap_interaction`  
- 数据：**`df_pred`（与训练分布一致）**，模型为校准后的 `best_model`。

### 5.3 因果（`03_causal`）

- **`get_estimate_causal_impact()`**（由 `config.CAUSAL_METHOD` 决定 XLearner / TLearner / CausalForestDML）  
- 数据：**`df_causal_use`（即插补后队列 `df_a/b/c`）**  
- 主干预列：`TREATMENT_COL`（默认 `exercise`）  
- 结局：`is_comorbidity_next`  
- 输出：`CAUSAL_ANALYSIS_*.csv`、`ATE_CI_summary_*.txt`、重叠/平衡/E-value、`PSM/PSW` 对比（模块内调用）等。  
- **Bootstrap**：ATE/ATT CI 在 fallback 时为 **按 `ID` 的 cluster bootstrap**（200 次）。  
- **后续**（非 `pool_only`）：敏感性、`ITE` 分层验证、列线图、**时序分析**（数据同为 **`df_causal_use`**）。

### 5.4 临床与外部验证（`04_eval`、`04b_external_validation`、`05_decision`）

- **`run_clinical_evaluation`、`run_clinical_decision_support`**：数据 **`df_pred`**（与模型训练一致）。  
- **`run_external_validation`**：数据 **`df_pred`**，冠军模型。

### 5.5 剂量反应（`04c_dose_response`）

- **`run_dose_response(df_causal_use, ...)`**（插补后表，避免剂量变量大量缺失导致不稳定）。

### 5.6 亚组（`06_subgroup`）

- **`df_for_subgroup = res_df`（含 `causal_impact_*`）若存在，否则 `df_causal_use`**  
- `run_subgroup_analysis`

### 5.7 返回值

- **发病率**：`df_causal_use[TARGET_COL].mean()`  
- **ATE 三元组**、**冠军 AUC**：供汇总图使用。

## 九、论文叙事建议（与方法对应）

- **预测**：预处理保留缺失 → **Pipeline 内 IterativeImputer**；对真连续变量另经 **StandardScaler**；**`ORDINAL_COUNT_IMPUTE_ONLY`**（`adlab_c`、`iadl`、`family_size`）**仅插补、不缩放**。划分：**GroupKFold / 时间划分**；阈值 **Youden 内验**；性能 CI **cluster bootstrap by ID**；冠军可选 **Platt 校准**。  
- **因果**：可写明使用 **单次完成数据集** `step1_imputed_full`（bulk 方法见该次 `methodology_justification.txt` / NRMSE 选优），与 **XLearner + PSM/PSW** 等；**不得**再声称 CPM 仅用 bulk 表且无 CV 内插补。  
- **Table 1 / 流失**：**插补前** `df_pre`，符合 STROBE 常见写法。

## 十一、核心变量字典（与代码一致）

### 11.1 `config.py` 中的分析锚点列

| 符号 | 默认值 | 含义 |
|------|--------|------|
| `TARGET_COL` | `is_comorbidity_next` | 监督学习与多数评价的主结局：下一波是否抑郁–认知共病 |
| `TREATMENT_COL` | `exercise` | 主因果分析的处理变量（0/1） |
| `COLS_TO_DROP` | `rgrip`, `grip_strength_avg`, `psyche`, `puff` | 读入插补表或构造预测用表时**整列删除**（高缺失或与主分析定义冲突） |
| `baseline_group` | 0/1/2 | 0=A 健康基线，1=B 仅抑郁，2=C 仅认知受损（**非 config 常量**，由预处理/重算写入） |

### 11.2 预处理 `preprocess_charls_data` 写入或强依赖的列

**输入 CSV 必需**：`ID`，`wave`；需能解析出 CES-D 列（如 `cesd10`）与认知总分列（`total_cognition` 或 `total_cog`）。

**核心衍生列（逻辑顺序）**  
- `is_depression`：CES-D ≥ `CESD_CUTOFF`（默认 10）。  
- `is_cognitive_impairment`：认知总分 ≤ `COGNITION_CUTOFF`（默认 10）。  
- `is_comorbidity`：抑郁与认知受损**同时**成立。  
- `next_wave_val` 及对 `is_depression` / `is_cognitive_impairment` / `is_comorbidity` / `fall_down` 的 `*_next` 或 `is_fall_next`：仅当 `next_wave_val == wave + 1` 时有效，否则 `NaN`。  
- **`is_comorbidity_next`**：主结局；随后 `dropna(subset=['is_comorbidity_next'])`。  
- `had_comorbidity_before`：组内历史是否曾共病；再筛 **`(is_comorbidity==0) & (had_comorbidity_before==0)`** 得入射队列。  
- **`baseline_group`**：见上表。  
- `chronic_burden`：七种慢病列之和（`hibpe`…`arthre` 中存在的列）。  
- `is_socially_isolated`：由 `marry` 与 `family_size` 规则构造（`family_size` 缺失用其中位数填充后再判）。  
- `bmi`：若存在 `mweight`/`mheight` 则重算，并 `clip(15,50)`。  
- `income_total`：对非负部分 `log1p`（缺失保留给后续 Pipeline）。  
- **`exercise`**：**缺失填 0**（与因果里 `T.fillna(0)` 叙述一致；完整病例敏感性另见 `run_sensitivity_scenarios`）。  

**写出文件（`write_output=True` 时）**：`preprocessed_data/CHARLS_final_preprocessed.csv`，`preprocessed_data/attrition_flow.csv`，可选 `preprocessing_missing_summary.csv`。最终保留列由代码中 `semantic_features` 与数据交集决定（非「全原始列」）。

### 11.3 `prepare_exposures`（`scripts/run_multi_exposure_causal.py`）

| 输入列 | 新增/修改列 | 规则 |
|--------|-------------|------|
| `sleep` | `sleep_adequate` | `sleep` 限制在 0–24 h 后 ≥6 为 1，否则 0；`sleep` 为 `NaN` 时 `sleep_adequate` 为 `NaN` |

主流程在插补表与 `df_pre` 上均可调用；之后在预测特征中 **`sleep_adequate` 仍被 `EXCLUDE_COLS_BASE` 排除**（预测用连续 `sleep`）。

### 11.4 `reapply_cohort_definition(df, cesd_cutoff, cognition_cutoff)`

**必需列**：`cesd*`、`total_cog*`/`total_cognition*`、`is_comorbidity_next`。  
**重算**：`is_depression`、`is_cognitive_impairment`、`is_comorbidity`、`had_comorbidity_before`，再应用与预处理相同的入射队列筛选与 **`baseline_group`**。  
用于：插补表入组、预测用表 `df_pa` 与 `df_clean` 对齐、截断值敏感性等。

### 11.5 监督学习特征矩阵 `compare_models`（`modeling/charls_model_comparison.py`）

- **调用**：`get_exclude_cols(df, target_col=TARGET_COL)` — **不传 `treatment_col`**。  
- **结果**：`TREATMENT_COL`（如 `exercise`）若仍为数值型且未被泄露关键字误伤，**可作为预测特征进入 X**（与因果中「处理变量不进协变量」不同）。  
- **Y**：`df[TARGET_COL].astype(int)`。  
- **分组**：划分与 bootstrap 用 **`df['ID']`**；时间划分用 **`df['wave']`**。  
- **X**：`df` 中**未被排除**且 **`select_dtypes(include=[np.number])` 的列**；即仅数值特征，**不含**需 LabelEncoder 的字符串列（预处理已把多数分类列编码为数值者除外）。

**`get_exclude_cols` 要点**（`utils/charls_feature_lists.py`）：  
- 基准排除：`EXCLUDE_COLS_BASE`（含 `ID`、`wave`、`is_comorbidity_next`、各类 `*_next` 泄露相关、`baseline_group`、`had_comorbidity_before`、`sleep_adequate` 等）。  
- 列名命中 **`LEAKAGE_KEYWORDS`**（如含 `cesd`、`score`、`cognition`…）的列一律排除。  
- 因果调用时传入 **`treatment_col`**：处理列及由其衍生的互斥列（如 `bmi_normal` 时排除 `bmi`）会排除；所有 `causal_impact_*` 前缀列排除。

**Pipeline 内数值处理**（`utils/charls_sklearn_preprocess_pipelines.py`）：  
- `CONTINUOUS_FOR_SCALING` 中列：**IterativeImputer → StandardScaler**。  
- **`adlab_c`、`iadl`、`family_size`**（`ORDINAL_COUNT_IMPUTE_ONLY`）：**仅 IterativeImputer**（与上条互斥，不在 `CONTINUOUS_FOR_SCALING` 内）。  
- 其余数值列：**仅 IterativeImputer**。  
- 分类名义变量 **`CATEGORICAL_NO_SCALE`** 不得出现在 `CONTINUOUS_FOR_SCALING`（断言防错）。

### 11.6 因果协变量矩阵 `estimate_causal_impact` / XLearner 路径（`causal/charls_recalculate_causal_impact.py`）

- **Y**：`is_comorbidity_next`（缺失行剔除）。  
- **T**：`treatment_col`；**`T.fillna(0)`**。  
- **X**：`get_exclude_cols(df_sub, target_col=Y, treatment_col=T)` 后剩余数值列；再经 **`fit_transform_numeric_train_only`**（训练子集 fit Imputer/Scaler）。  
- **聚类**：优先 `communityID`，否则 `ID`（用于部分估计器的群组结构）。

### 11.7 Table 1 呈现的变量域（`data/charls_table1_stats.BPS_SECTIONS`）

按生物–心理–社会块组织，主要包括：  
`age`, `bmi`, `mwaist`, `gender`, `fall_down`, `disability`, `pulse`, `systo`, `diasto`, `lgrip`, `wspeed`, 七种慢病, `adlab_c`, `iadl`, `srh`, `satlife`, `rural`, `is_socially_isolated`, `pension`, `ins`, `retire`, `edu`, `marry`, `family_size`, `income_total`, `exercise`, `drinkev`, `smokev`, `sleep_adequate`, `sleep`，以及定义块中的 CES-D / 认知分 / `baseline_group` 等（以表中 `defining_*` 配置为准）。

### 11.8 Table 1b 发病密度（`data/charls_incidence_density.py`）

**输入列**：`wave`，`is_comorbidity_next`，`baseline_group`。  
**输出文件**：`OUTPUT_ROOT/table1b_incidence_density.csv` 与 `results/tables/table1b_incidence_density.csv`（列名：`Baseline_Cohort`, `Person_Wave_Observations`, `Total_Person_Years`, `Incident_Cases`, `Incidence_Rate_per_1000_PY`）。
